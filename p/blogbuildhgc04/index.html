行动指南

本文所进行的博客美化是基于Stack主题v3.26.0上进行的代码修改，不同的版本以及不同的主题在细节处理上可能会有稍许不同，如果您是初次搭建且此前无代码基础，您可以选择和本文一样的主题版本进行相关美化用于练手。

博客的优化与美化是无穷无尽的，在之后您或许还会接触到魔改版——简单来说即在原主题基础上进行底层代码的变动或是默认添加一些功能，如果您已经有了对博客的优化美化需求，建议您先将需求进行简单的分类，再按照您的需求从急到缓依次解决，例如：

  • 针对核心功能的优化，例如字体、主题背景、侧边栏属性。
  • 辅助功能的添加，例如数据统计、头图欢迎、页脚动图。

并建议您如果存在多处地方需要调整，先备份一份可以正常执行的源代码博客数据，并且在每次进行修改后及时用本地浏览功能测试是否可以正常运行（命令hugo server -D），减少因多次修改出现某处代码发生不明错误而难以具体排查的可能。

Hugo采用的是配置覆盖的形式，即优先采取您在博客主目录下所进行的配置，再采取themes文件夹下您所设置生效的主题中的配置，建议您尽量在主目录下进行修改，原始状态下主目录文件夹多为空文件夹，您可以复制themes文件夹下主题中对应的文件夹，再进行修改。

在这一章我们解决怎么做的问题，并通过反复的训练您可能会发现有几个文件是我们经常修改的地方，但本章我们尽量不讨论为什么我这样做就能实现这个效果的涉及代码理论的问题，而是先去实现它。

     * 滚动回顶部初始化
     */
    function initScrollTop() {
        let rightSideBar = document.querySelector(".right-sidebar");
        if (!rightSideBar) {
            return;
        }
        // 添加返回顶部按钮到右侧边栏
        let btn = document.createElement("div");
        btn.id = "backTopBtn";
        btn.onclick = backToTop
        rightSideBar.appendChild(btn)
        // 滚动监听
        window.onscroll = function() {
            // 当网页向下滑动 20px 出现"返回顶部" 按钮
            if (document.body.scrollTop > 20 || document.documentElement.scrollTop > 20) {
                btn.style.display = "block";
            } else {
                btn.style.display = "none";
            }
        };
    }

    /**
     * 返回顶部
     */
    function backToTop(){
        window.scrollTo({ top: 0, behavior: "smooth" })
    }

    initScrollTop();
</script>

方案选择

为博客搭建一个评论区有多种方案，每种方案都有自己的优势劣势，并且随着技术迭代发展，会有更多的方案出现与退出，建议您根据自己搭建博客的方式来选择最适合的方案，本文选择的是giscus。

giscus方案

👉 官方引导页面 https://giscus.app/zh-CN

giscus 加载时，会使用  GitHub Discussions 搜索 API  根据选定的映射方式（如 URL、pathname、<title> 等）来查找与当前页面关联的 discussion。如果找不到匹配的 discussion，giscus bot 就会在第一次有人留下评论或回应时自动创建一个 discussion。 访客如果想要评论，必须按照 GitHub OAuth 流程授权  giscus app   代表他发布 ，或者可以直接在 GitHub Discussion 里评论。你可以在 GitHub 上管理评论。

您可能其中的一些名词稍感到困惑，但我们要使用它而并非研究其原理，只需要提取出其中的关键词，事实上这在引导页面中的步骤设置中均有体现：

  • 它是基于GitHub自带的discussion功能，因此我们需要有GitHub上的账号以及仓库；
  • 它进行了查找与关联的工作，因此它需要能够访问到我们的仓库，这指向我们的仓库应当是公开访问的；
  • giscus并非GitHub的原生功能，我们需要能够使用giscus，需要进行下载和配置；
  • 除此之外，我们需要让我们的博客能够加载评论区，需要对博客数据进行修改；
